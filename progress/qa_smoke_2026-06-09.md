## Bugs encontrados

### Bug 1 — BLOCKER (datos de prueba) — seed con UUIDs RFC-inválidos
`progress/seed_test_users.sql` usa ids tipo `11111111-1111-1111-1111-111111111111` (variante RFC inválida). Consecuencias:
- Backend (`@IsUUID`) rechaza cualquier POST que incluya ese `sellerId` → seller1 NO podía crear clientes desde UI (`400: sellerId must be a UUID`).
- Los INSERT de clients/tasks/activities/deals del seed nunca aplicaron → seller1 estaba sin datos.

**Reparado en DB** (no en el archivo seed):
```sql
INSERT INTO sellers (id, name, profile, user_id, active) VALUES
('a1b2c3d4-0000-4000-8000-000000000001', 'Juan Perez QA', 'QA test seller', '44444444-4444-4444-4444-444444444444', true);
UPDATE users SET seller_id='a1b2c3d4-0000-4000-8000-000000000001' WHERE username='seller1';
DELETE FROM sellers WHERE id='11111111-1111-1111-1111-111111111111';
```
**Pendiente**: corregir seed_test_users.sql para usar UUIDs v4 válidos (users `33333333-...`, `44444444-...`, `55555555-...` también son inválidos; funcionan para login pero fallarían en cualquier validación @IsUUID).

### Bug 2 — MAJOR (UX) — alta de cliente falla silenciosamente
Si POST /api/clients devuelve 400, el modal "Nuevo cliente" no muestra ningún error: el usuario cree que guardó y no pasó nada.
Repro: como seller con sellerId inválido (pre-fix), llenar form y Guardar → 400 en consola, UI muda.

### Bug 3 — MINOR — opción "Facebook" en source no aceptada por backend
El combobox de fuente en "Nuevo cliente" incluye "Facebook", pero el DTO solo acepta: Prospección propia, Cliente existente, Referido, Expo, Marketing, LinkedIn, Web, Dirección Comercial. Elegir Facebook → 400 (y por Bug 2, silencioso).

### Bug 4 — MINOR — "Salir" no redirige de inmediato
Click en Salir limpia la sesión pero la vista actual queda renderizada; el redirect a /login ocurre hasta la siguiente navegación. Repro: cualquier rol → Salir → la página no cambia.

### Bug 5 — MINOR — 403 en consola con rol Seller
Con sesión Seller el frontend llama `GET /api/sellers` y `GET /api/settings` (admin/director-only) → errores 403 en consola en /clientes, /configuracion y /coaching. No rompe funcionalidad (hay fallbacks), pero ensucia consola.

### Observaciones (confirmar diseño)

- Sidebar muestra un combobox "Rol" (Administrador/Gerente/Representante) visible en todos los roles — parece artefacto de demo del HTML standalone.
